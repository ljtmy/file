# 约定式提交工具链部署指南

## 0. 前置准备

确保系统已安装 Node.js，且当前目录已经是一个 Git 仓库（已执行过 `git init`）。

## 1. 安装核心依赖包

在仓库根目录执行以下命令，初始化 npm 环境并安装所需的开发依赖：

```bash
# 初始化 package.json
npm init -y

# 安装交互式提示工具
npm install commitizen cz-customizable --save-dev

# 安装规范校验工具
npm install @commitlint/cli commitlint-config-cz --save-dev

# 安装 Git 钩子工具
npm install husky --save-dev
```

## 2. 配置黑名单 (.gitignore)

```plaintext
# Node.js 依赖
node_modules/

# 常见的底层开发编译产物
build/
*.o
*.out
*.exe

# 操作系统自动生成的隐藏文件
.DS_Store
Thumbs.db
```

## 3. 配置 package.json

打开根目录的 `package.json`​，在已有的 JSON 结构中增加 `scripts`​ 快捷命令和 `config` 适配器路径。修改后的关键部分应如下所示：

JSON

```
{
  "name": "my-research-workspace",
  "version": "1.0.0",
  "description": "科研日志、学习笔记与底层技术实验归档",
  "scripts": {
    "commit": "git-cz",
    "prepare": "husky"
  },
  "devDependencies": {
    "@commitlint/cli": "^19.0.0",
    "commitizen": "^4.3.0",
    "commitlint-config-cz": "^0.13.3",
    "cz-customizable": "^7.0.0",
    "husky": "^9.0.0"
  },
  "config": {
    "commitizen": {
      "path": "node_modules/cz-customizable"
    }
  }
}
```

## 4. 创建交互菜单配置 (.cz-config.js)

在根目录新建 `.cz-config.js` 文件，配置自定义的 10 大核心类型和作用域。

JavaScript

```
module.exports = {
  types: [
    { value: 'log',      name: 'log:      [日志] 记录工作进度、周报或实验过程' },
    { value: 'note',     name: 'note:     [笔记] 技术调研、源码解析或论文阅读' },
    { value: 'outcome',  name: 'outcome:  [成果] 提交正式报告、演示文档或产出物' },
    { value: 'exp',      name: 'exp:      [实验] 编写验证性原型、测试脚本或Demo' },
    { value: 'perf',     name: 'perf:     [性能] 底层优化、降低延迟或提升吞吐量' },
    { value: 'fix',      name: 'fix:      [修复] 修正笔记错误或实验脚本中的漏洞' },
    { value: 'refactor', name: 'refactor: [重构] 优化代码结构或重新整理笔记小节' },
    { value: 'build',    name: 'build:    [构建] 调整编译配置(CMake/Makefile)或依赖' },
    { value: 'docs',     name: 'docs:     [文档] 更新README、环境指南或项目说明' },
    { value: 'chore',    name: 'chore:    [杂项] 仓库日常维护、忽略文件或配置变更' }
  ],
  scopes: [
    { name: 'ebpf' },
    { name: 'linux' },
    { name: 'dpdk' },
    { name: 'c++' },
    { name: 'crypto' },
    { name: 'sys-monitor' }
  ],
  allowCustomScopes: true, // 允许选择 custom (自定义)
  allowEmptyScopes: true,  // 允许选择 empty (跳过/空)
  messages: {
    type: '选择提交类型:',
    scope: '选择/输入范围(可选):',
    customScope: '请输入自定义的范围:',
    subject: '简短描述修改 (必填):',
    body: '详细背景说明 (可选，使用 "|" 换行):',
    confirmCommit: '确定提交以上信息？(y/n)'
  },
  skipQuestions: ['footer', 'breaking'],
  subjectLimit: 100
};
```

## 5. 创建拦截规则配置 (commitlint.config.js)

在根目录新建 `commitlint.config.js`，确保校验工具与上方配置的类型保持一致：

JavaScript

```
module.exports = {
  extends: ['cz'],
  rules: {
    'type-enum': [
      2,
      'always',
      [
        'log', 'note', 'outcome', 'exp', 'perf', 
        'fix', 'refactor', 'build', 'docs', 'chore'
      ]
    ],
    'type-empty': [2, 'never'],
    'subject-empty': [2, 'never']
  }
};
```

## 6. 日常使用工作流

部署完成后，标准的提交动作将变更为：

1. ​**暂存文件**​：`git add .`
2. ​**唤起菜单**​：`npm run commit` （根据终端中文提示选择类型、作用域并填写描述）
3. ​**推送到远端**​：`git push`
